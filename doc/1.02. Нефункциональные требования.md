# Нефункциональные требования

* Масштабируемость в зависимости от количества пользователей и объявлений, балансирование нагрузки
* Доступность системы 99,95% (uptime)
* Наличие подключенных инструментов наблюдаемости (observability), мониторинг, логирование, трассировка для разбора ошибок
* Обеспечение быстрого ответа на запросы пользователей, меньше 3 секунд
* Наличие инструментов хранения больших объемов данных с системой кэширования
* Защита данных пользователей от несанкционированного доступа, обеспечение безопасности во время проведения платежей
* Обеспечение отсутствия мошенников, мошеннических объявлений, несанкционированного доступа к платежным данным
