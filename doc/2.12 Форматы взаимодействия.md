# Форматы взаимодействия

Взаимодействие разбито на три слоя: client layer (пользовательский слой), edge layer ("пограничный" уровень) и services
(сервисный слой).

Client Layer <-> Edge Layer:
Для MVP будет реализовано только APP клиент через который пользователи смогут взаимодействовать с приложением. 
Основной протокол HTTP/1.x, после успешного запуска рассмотреть переход на целевое взаимодействие HTTP/2.
Функция чата будет реализована посредством WebSocket-взаимодействия, так как общение между продавцов и покупателем
должно быть оперативным.
Push-уведомления получаем по HTTP/1.x, интегрируясь с FCM (для Android) или APNs (для iOS).

Edge Layer <-> Service Layer:
Основным протоколом выступает GRPC. Формат взаимодействия между API Gateway и нашими сервисами будет стандартизирован,
позволяет легко версионировать будущие изменения в контрактах. Более того, объем одного сообщения по сравнению с HTTP ниже,
тем самым снижаем объем передаваемого трафика.

Service Layer
Основным протоколом выступает HTTP/2. Сервисы развернуты в кластере Kubernetes, взаимодействие через локальные HTTP-урлы.
Взаимодействие синхронное. Работа с сервисами аналитики и сервисом уведомления асинхронное, так как не входят в основной перечень
критически важных сервисов.
Пример RestApi в формате в директории proto (для взаимодействия API Gateway -> Services Layer).

## Service Discovery:
Когда микросервисы развернуты в динамической среде (например, в Kubernetes или Docker), их IP и порты могут меняться.
Service Discovery решает эту проблему:

Регистрация сервиса: при запуске микросервис регистрируется в центральном реестре сервисов (service registry),
указывая своё имя, IP, порт и статус.
Обнаружение сервиса: другой микросервис запрашивает у реестра актуальный адрес нужного сервиса по имени.
Балансировка нагрузки: клиент может выбирать один из нескольких экземпляров сервиса.
Здоровье сервиса: реестр проверяет "живость" сервисов (через health checks) и удаляет неработающие.

## API Gateway
Настройка API Gateway — ключевой элемент архитектуры микросервисов. Он выступает как единая точка входа, обеспечивая:
Маршрутизацию запросов к нужным микросервисам
Аутентификацию и авторизацию
CORS
Ограничение скорости (Rate Limiting, Circuit Breaker)
Логирование, мониторинг, кэширование
