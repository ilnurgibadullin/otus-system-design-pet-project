# Система хранения данных

Основные системы хранения данных:
1. PostgreSQL
2. Redis
3. S3
4. CDN

PostgreSQL используется в качестве основного инструмента хранения данных - профиля, объявлений, информаций об оплате,
доставка, сообщений в чате.
Для кэширования - Redis.
Информация для кэширования:
1. Результатов поиска.
2. Страниц объявлений
3. Структуры категорий
4. Информация о пользователях

Планируется, что вся информация в первую очередь записывается в PostgreSQL, асинхронно кэшируется в Redis.

CDN может значительно повысить производительность и надежность системы.
Список ресурсов, которые следует распределять через CDN:

1. Статические медиафайлы
   Изображения товаров/объявлений: JPG, PNG, GIF, SVG, WebP.
   Видео и аудио: Обзорные видео, демонстрации товаров.
2. CSS и JavaScript файлы
   Стили (CSS): Основные стили сайта, темы оформления.
   Скрипты (JS): Библиотеки (jQuery, React, Vue.js), пользовательские скрипты интерфейса.
3. Шрифты
   Пользовательские шрифты (WOFF, WOFF2, TTF, EOT).
4.Манифест-файлы и метаданные
   PWA манифесты.
   Фавиконки (favicon.ico, apple-touch-icon.png и др.).
5. Документы
   Условия использования, политика конфиденциальности, инструкции по оплате и доставке.

Принципы работы с CDN для обеспечения низкой задержки и высокой доступности:
1. Правильная классификация контента - статический контент (через CDN) и динамический контент (с основного сервера).
2. Оптимизация кэширования - настройка ttl и версионирование ресурсов
3. Географическое распределение - выбор расположения (Москва, Санкт-Петербург, Краснодар, Новосибирск)
4. Автоматическая оптимизация изображений и медиа
5. Защита от DDoS
6. Мониторинг, отслеживание производительности и health-check
7. Резервные механизмы и отказоустойствость
